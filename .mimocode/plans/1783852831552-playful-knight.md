# P2: Агент умеет билдить и прогонять тесты

## Context

Агент (codegen-like) должен уметь:
1. Сгенерировать изменение
2. Запустить build
3. Запустить tests
4. Увидеть ошибку
5. Исследовать причину
6. Исправить
7. Повторить проверки

Playground работает: `https://gpu-0.devops-playground.innopolis.university`
Модели с tools support: `qwen2.5-coder:32b`, `qwen3:32b`, `qwen3:14b`

## Архитектура

```
review-agent (Mastra, TS)
    │
    ├── tool: run-command(command, cwd)
    │       ↓
    │   sandbox container (Docker)
    │       ├── Go toolchain
    │       ├── Node.js
    │       ├── ripgrep
    │       └── linters (golangci-lint, eslint)
    │
    ├── tool: read-file(path)
    ├── tool: write-file(path, content)
    ├── tool: search-code(query)
    └── tool: list-directory(path)
```

## Что реализуем

### 1. Sandbox container

Docker image с инструментами:
```dockerfile
FROM golang:1.22-alpine
RUN apk add --no-cache nodejs npm ripgrep git
RUN go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest
WORKDIR /workspace
```

Запускается как Docker container, в который mount'ится код из PR.

### 2. Sandbox API (Go service)

Маленький Go сервис который:
- Принимает HTTP запросы с командами
- Выполняет их в sandbox container
- Возвращает stdout/stderr

Эндпоинты:
- `POST /exec` — выполнить команду, вернуть результат
- `POST /write` — записать файл
- `GET /read` — прочитать файл

### 3. Tools для Mastra агента

| Tool | Описание | Реализация |
|------|----------|------------|
| `run-build` | Запустить build проекта | HTTP → sandbox API → `go build ./...` или `npm run build` |
| `run-tests` | Запустить тесты | HTTP → sandbox API → `go test ./...` или `npm test` |
| `run-lint` | Запустить linter | HTTP → sandbox API → `golangci-lint run` или `npm run lint` |
| `read-file` | Прочитать файл из sandbox | HTTP → sandbox API → cat файла |
| `write-file` | Записать файл в sandbox | HTTP → sandbox API → write |
| `search-code` | Поиск по коду | HTTP → sandbox API → ripgrep |

### 4. Workflow integration

Добавить в review-agent workflow новый step:
```
fetchContext → createPlan → investigate → [build & test] → verify → render
```

Шаг `build & test`:
- Агент запускает build/test через tools
- Если ошибки — добавляет их в контекст для investigate
- Если всё ок — пропускает

3. Sandbox container поднимается и выполняет команды
4. Агент может запустить build/test через tools
5. E2E: агент получает PR с ошибкой → находит ошибку через build →报告
