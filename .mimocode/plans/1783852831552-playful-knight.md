# P0: Прокидывание PR description в контекст review

## Что делаем

Добавляем описание PR (body) в контекст LLM при review. Сейчас модель видит только diff + AGENTS.md + README.md, но не знает ЗАЧЕМ этот PR. PR description содержит "what this PR does" — это критичный контекст для качественного review.

## Анализ

### Текущий flow

```
GitFlame webhook → review-consumer → LLM
                                 ↗
              diff + AGENTS.md + README.md
```

PR description **не передаётся** — она не в webhook payload (только action, number, head SHA).

### Что нужно

1. **GitFlame API**: добавить метод получения PR description
2. **Go review-consumer**: fetch description + передать в промпт
3. **TS review-agent**: fetch description + передать в workflow
4. **Промпты**: включить description в user message

### 4. TS: GitFlame client — getPRDescription

**Файл:** `packages/review-agent/src/services/gitflame-client.ts`

```typescript
interface PRDetails {
    body: string
}

async getPRDescription(owner: string, repo: string, pullNumber: number): Promise<string> {
    const pr = await this.requestWithRetry<PRDetails>(
        `/api/v1/repos/${owner}/${repo}/pulls/${pullNumber}`
    )
    return pr.body || ""
}
```
